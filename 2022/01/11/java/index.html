day1

  • 类变量，静态变量

    访问修饰符 static 数据类型 变量名

    通过 类名.变量名 来访问 没有创建对象实例也可以访问

    对象名.属性名

    类变量的访问遵守相关的访问权限

  • 类方法 静态方法

    访问修饰符 static 数据返回类型 方法名

    静态方法可以访问静态属性

    静态方法通可过类名调用

    类名.方法名 对象名.方法名

    类方法中不允许使用和对象相关的关键字 比如this super

    只能访问静态成员

  • 普通成员

    可以访问非静态成员，也可以访问静态成员

  • main

    java 虚拟机调用类的main方法，所以该方法必须public

    java 虚拟机运行时不必创建对象，所以该方法必须static

    该方法接受String数组 执行程序后带程序

    main是静态方法

    可以访问该类的静态成员

    不能访问该类的非静态成员

    在idea中传String[] args

    • 单例 单个实例 饿汉

  • final 关键字

    修饰的类不允许被继承

    修饰的方法不允许被子类重写，可以继承

    修饰的属性不允许被修改

    修饰的局部变量不允许被修改

    final修饰的属性又叫常量，一般用大写来命名

    final修饰的属性必须赋值 定义时，代码块中，构造器中

    static final 不能在构造器中 一个类加载，一个实例创建不一样

    使用final static 不会导致类的加载

    包装类 String 等 是final 不能被继承

  • 抽象类

    但父类的某些方法需要声明，却不需要如何具体实现 父类方法不确定问题

    该类方法为抽象方法，该类为抽象类

    抽象类被继承，由其子类来实现功能

    访问修饰符 + abstract + 类名 {}

    访问修饰符 + abstract + 方法名()

    设计模式使用居多

    抽象类不能实例化

    抽象类可以没有抽象方法，可以有普通方法

    abstract只能用来修饰属性和方法

    抽象类还是类，可以有任意成员

    如果一个类继承抽象类，则它必须实现抽象类的所有方法，或者声明自己为抽象类

    抽象方法不允许使用private final static 与重写相矛盾

    10000.for idea快捷键

  • 接口

    规范

    给出一些没有实现的方法，封装到一起，到某个类要使用时，在根据类使用时调用

    接口不能被实例化

    接口中的方法都是public

    接口中的抽象方法可以不用abstract修饰

    一个普通类实现接口就必须将该接口中的所有方法都实现 快捷键alt + enter

    抽象类实现接口时可以不用实现接口方法

    一个类可以同时实现多个接口

    接口中的属性都是final public static final

    访问接口属性 接口名.属性名

    接口不能继承其他类，但可以继承其他接口

    接口与接口之间是继承 接口与类是实现

  • 接口与类继承区别

    继承：单继承 extends 一个类

    接口：多继承 implement 多个接口

    可以 extends … implement ….

    弹幕一个有趣的说法：你只能有一个爸，但是你可以有多个师傅

    接口是对Java单继承的补充

    接口更加灵活

  • 接口的多态

    多态参数

day3

  • 内部类

    一个类的内部完整嵌套了另一个类结构，被嵌套的类称为内部类，嵌套其他类的类称为外部类

    类的五大成员：属性，方法，构造器，代码块，内部类

    内部类可以直接访问私有属性，并且可以体现类与类之间的包含关系

    四种内部类：

    定义在外部类局部位置上(比如方法内)

    1 局部内部类 有s类名 本质是类 定义在方法或代码块中 不能用修饰符但可以加final 作用域在定义方法体内 直接访问外部类成员 外部类在方法中可以创建内部类实例 然后调用方法即可 访问外部类的成员 用外部类名.this.变量名

    2 匿名内部类 本质是类，是内部类，该类没有名字(实则系统分配)，同时还是一个对象，在方法或代码块中 基本语法 运行类型，编译类型….面向对象 编译类型看=左边，运行类型是匿名内部类

    getClass 获取运行类型 匿名内部类使用一次，就不能再使用 生成对象可以反复使用

    定义在外部类的成员位置上

    1 成员内部类 定义在外部类的成员位置 作用域 同名 外部类.this

    2 静态内部类 static 也是成员

在代码中获取原始异常可以使用Throwable.getCause()方法。如果返回null，说明已经是“根异常”了。

在catch中抛出异常，不会影响finally的执行。JVM会先执行finally，然后抛出异常。

finally抛出异常后，原来在catch中准备抛出的异常就“消失”了，因为只能抛出一个异常。没有被抛出的异常称为“被屏蔽”的异常（Suppressed Exception）。
